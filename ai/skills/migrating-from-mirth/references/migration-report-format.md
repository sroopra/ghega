# Migration Report Format

This document describes the report format generated by the
`ghega migrate mirth` CLI command.

## Output Directory Structure

After running:

```
ghega migrate mirth <export-dir> --out <output-dir>
```

The output directory contains:

```
<output-dir>/
  migration-report.yaml          # Summary across all channels
  <channel-name-1>/
    channel.yaml                 # Auto-converted Ghega channel
    migration-report.yaml        # Per-channel report
    rewrite-tasks.yaml           # Typed rewrite tasks (may be empty)
  <channel-name-2>/
    ...
```

## Summary Report (`migration-report.yaml`)

| Field | Type | Description |
|-------|------|-------------|
| `generatedAt` | string (RFC3339) | When the reports were generated |
| `channels` | array | One `ChannelSummary` per channel |
| `totalChannels` | integer | Total number of channels processed |
| `totalAutoConverted` | integer | Channels with status `auto-converted` |
| `totalNeedsRewrite` | integer | Channels with status `needs-rewrite` or `mixed` |
| `totalUnsupported` | integer | Channels with status `unsupported` |

### ChannelSummary

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Sanitized channel name |
| `status` | string | `auto-converted`, `needs-rewrite`, `unsupported`, or `mixed` |
| `rewriteTasksCount` | integer | Number of items in `needsRewrite` |
| `warningsCount` | integer | Number of warnings during conversion |

## Per-Channel Report (`<channel>/migration-report.yaml`)

| Field | Type | Description |
|-------|------|-------------|
| `channelName` | string | Sanitized Ghega-compatible name |
| `originalName` | string | Original Mirth channel name |
| `status` | string | Overall migration status |
| `autoConverted` | array of `ConvertedItem` | Elements migrated automatically |
| `needsRewrite` | array of `RewriteTaskItem` | Patterns requiring manual rewrite |
| `unsupported` | array of `UnsupportedItem` | Features not yet supported |
| `warnings` | array of strings | Human-readable warnings |

### ConvertedItem

| Field | Type | Description |
|-------|------|-------------|
| `element` | string | Element type, e.g. `source_connector`, `destination_connector`, `mapping` |
| `description` | string | Human-readable description of the conversion |

### RewriteTaskItem

| Field | Type | Description |
|-------|------|-------------|
| `severity` | string | `low`, `medium`, or `high` |
| `description` | string | What the original code did and what replacement is needed |
| `category` | string | Pattern category, e.g. `conditional`, `loop`, `logger` |

### UnsupportedItem

| Field | Type | Description |
|-------|------|-------------|
| `feature` | string | Feature category, e.g. `e4x_manipulation`, `external_call` |
| `description` | string | Explanation of why it is unsupported |

## Status Derivation

- `auto-converted`: no rewrite tasks and no unsupported items
- `needs-rewrite`: at least one rewrite task, no unsupported items
- `unsupported`: at least one unsupported item, no rewrite tasks
- `mixed`: both rewrite tasks and unsupported items are present

## Rewrite Tasks File (`<channel>/rewrite-tasks.yaml`)

This file contains a YAML array of `RewriteTaskItem` objects. It is always
written even if empty, so that downstream tooling can assume the file exists.

## Example Report Snippet

```yaml
channelName: adt-a01-feed
originalName: ADT A01 Feed
status: needs-rewrite
autoConverted:
  - element: source_connector
    description: 'Source connector mapped to type "mllp"'
  - element: destination_connector
    description: 'Destination connector mapped to type "http"'
needsRewrite:
  - severity: high
    description: 'Replace conditional with an equivalent mapping or custom transformation.'
    category: conditional
unsupported: []
warnings:
  - 'JavaScript transformers/filters are present but not converted in this step'
```

## Using Reports Programmatically

The summary report is useful for dashboards and CI pipelines. The per-channel
reports and rewrite-tasks files are the inputs for the
`writing-typed-rewrite-tasks` skill.
