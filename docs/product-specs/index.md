# Product Specifications

Product specs describe user-facing features and behaviors. Each spec defines what the feature does, who it serves, and acceptance criteria.

## Specifications

_No product specs have been written yet. As features mature beyond ADR stage, document them here._

## Template

When creating a new product spec, include:

1. **Title** — feature name
2. **Status** — Draft / In Review / Accepted
3. **User story** — who benefits and how
4. **Acceptance criteria** — testable conditions for completion
5. **Design references** — links to relevant ADRs or design docs
6. **Out of scope** — what this spec explicitly does not cover
